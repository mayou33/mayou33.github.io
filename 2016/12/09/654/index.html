####

CentOS 6&nbsp; vsftpd安装

&nbsp;要求：不允许匿名登陆，只允许ftp用户登陆，并指定ftp的固定位置。

用户：zhangsan&nbsp; 固定位置/data/ftp/zhangsan

用户：lisi&nbsp; 固定位置/data/ftp/lisi

###

先关闭SELinux及iptables防火墙

iptables -I INPUT -p tcp --dport 20 -j ACCEPT

iptables -I INPUT -p tcp --dport 21 -j ACCEPT

&nbsp;

vsftpd提供三种登录形式:

1)anonymous(匿名帐号)

使用anonymous是最常用的。如果用户在FTP上没有帐号，那么用户可以以anonymous为用户名，电子邮件地址为密码进行登录。匿名ftp服务器的根目录是/var/ftp，所以为了减轻ftp的负载，anonymous帐号的权限很小，一般只读。

2)real

real 帐号称本地帐号，类似于windows下的&#39;管理员&#39;。用户必须以真实的用户名和密码进行登录，前提条件是在FTP上拥有帐号。当使用真是帐号登录后，登录目录就是该用户自己的目录，该目录在系统建立帐号，系统自动建立。真实 用户可以访问整个目录结构，除非万不得已，应避免。

3)guest

该帐号的权限仅仅限于文件传输，guest是一种弱化了的real帐户，弱化在于guest登录之后，不能访问主目录以外的内容。

一：安装vsftpd

查看是否已经安装vsftpd

rpm -qa | grep vsftpd

如果没有，就安装，并设置开机启动

yum -y install vsftpd ftp

chkconfig --level 2345 vsftpd on

配置文件：/etc/vsftpd/vsftpd.conf&nbsp;&nbsp;&nbsp;

&nbsp;默认存储路径/var/ftp

对于ftp这个用户的管理，我们应该查看/etc/passwd ，然后修改ftp用户那行；

ftp:x:14:50:FTP User:/var/ftp:/sbin/nologin

2.配置vsftpd

编辑/etc/vsftpd/vsftpd.conf文件，配置vsftp服务：

mv /etc/vsftpd/vsftpd.conf /etc/vsftpd/vsftpd.conf.bak

touch /var/log/vsftpd.log

vim /etc/vsftpd/vsftpd.conf

&nbsp;##

anonymous_enable=NO

#设定不允许匿名访问

local_enable=YES&nbsp;

#设定本地用户可以访问。注意：主要是为虚拟宿主用户，如果该项目设定为NO那么所有虚拟用户将无法访问。

write_enable=YES

#设定可以进行写操作。

local_umask=022&nbsp;

#设定上传后文件的权限掩码

anon_upload_enable=NO&nbsp;&nbsp;

#禁止匿名用户上传。

anon_mkdir_write_enable=NO

#禁止匿名用户建立目录

dirmessage_enable=YES

#设定开启目录标语功能。

xferlog_enable=YES&nbsp;

#设定开启日志记录功能。

connect_from_port_20=YES

#设定端口20进行数据连接。

chown_uploads=NO&nbsp;

#设定禁止上传文件更改宿主。

chroot_local_user=YES&nbsp;

#设定登陆后.只可以访问自己的属主目录.不可访问上一层目录文件

xferlog_file=/var/log/vsftpd.log&nbsp;

#设定Vsftpd的服务日志保存路径。注意，该文件默认不存在。必须要手动touch出来，并且由于这里更改了Vsftpd的服务宿主用户为手动建立的Vsftpd。必须注意给与该用户对日志的写入权限，否则服务将启动失败。

xferlog_std_format=YES&nbsp;

#设定日志使用标准的记录格式。

#（待定）nopriv_user=xxx&nbsp; //设定支撑Vsftpd服务的宿主用户为手动建立的xxx用户。注意，一旦做出更改宿主用户后，必须注意一起与该服务相关的读写文件的读写赋权问题。比如日志文件就必须给与该用户写入权限等

async_abor_enable=YES

#设定支持异步传输功能。

ascii_upload_enable=YES

&nbsp;#设定支持ASCII模式的上传功能。

ascii_download_enable=YES

#设定支持ASCII模式的下载功能。

ftpd_banner=Welcome to blah FTP service

#设定Vsftpd的登陆标语。

chroot_list_enable=NO&nbsp;&nbsp;

#禁止用户登出自己的FTP主目录。

ls_recurse_enable=NO&nbsp;

#禁止用户登陆FTP后使用&quot;ls -R&quot;的命令。该命令会对服务器性能造成巨大开销。如果该项被允许，那么挡多用户同时使用该命令时将会对该服务器造成威胁。

listen=YES&nbsp;

#设定该Vsftpd服务工作在StandAlone模式下。顺便展开说明一下，所谓StandAlone模式就是该服务拥有自己的守护进程支持，在ps -A命令下我们将可用看到vsftpd的守护进程名。如果不想工作在StandAlone模式下，则可以选择SuperDaemon模式，在该模式下vsftpd将没有自己的守护进程，而是由超级守护进程Xinetd全权代理，与此同时，Vsftp服务的许多功能将得不到实现。

pam_service_name=vsftpd

&nbsp;#设定PAM服务下Vsftpd的验证配置文件名。因此，PAM验证将参考/etc/pam.d/下的vsftpd文件配置。

userlist_enable=YES

&nbsp;#设定userlist_file中的用户将不得使用FTP。

tcp_wrappers=YES

#设定支持TCP Wrappers

userlist_deny=NO

userlist_file=/etc/vsftpd/user_list

pasv_enable=YES

use_localtime=YES

#####end#####

添加ftp用户，命令格式： #adduser -d /目录路径 -g ftp -s /sbin/nologin 用户名

#adduser -d /data/ftp/zhangsan&nbsp; -g ftp -s /sbin/nologin zhangsan

#adduser -d /data/ftp/lisi&nbsp; -g ftp -s /sbin/nologin lisi

增加了一个名为zhangsan 的用户，它的目录属于/data/zhangsan下面，属于ftp用户组。此用户是不可以登陆我们终端服务器的。

添加密码：

#passwd zhangsan

#passwd lisi

设置用户名为FTPUSER的用户在目录下可以读写的权限

touch /data/ftp/zhangsan/test-z.txt

touch /data/ftp/lisi/test-l.txt

&nbsp;chown -R zhangsan:ftp /data/ftp/zhangsan

&nbsp; chown -R lisi:ftp /data/ftp/lisi

cp /etc/vsftpd/user_list /etc/vsftpd/user_list.bak

vi /etc/vsftpd/user_list
