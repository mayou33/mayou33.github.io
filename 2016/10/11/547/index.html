mysql只读模式的设置方法

2016-10-11 mysql mysql

  在MySQL数据库中，在进行数据迁移和从库只读状态设置时，都会涉及到只读状态和Master-slave的设置和关系。

&nbsp; &nbsp; &nbsp; &nbsp;经过实际测试，对于MySQL单实例数据库和master库，如果需要设置为只读状态，需要进行如下操作和设置：

&nbsp; &nbsp; &nbsp; 将MySQL设置为只读状态的命令：

# mysql -uroot -p

mysql&gt; show global variables like &quot;%read_only%&quot;;

mysql&gt;<span style="color: rgb(255, 0, 0); background-color: inherit;"><span style="background-color: inherit;">&nbsp;</span>flush tables with read lock;</span>

mysql&gt;<span style="color: rgb(255, 0, 0); background-color: inherit;"><span style="background-color: inherit;">&nbsp;</span>set global read_only=1;</span>

mysql&gt; show global variables like &quot;%read_only%&quot;;

将MySQL从只读设置为读写状态的命令：

mysql&gt;<span style="background-color: inherit;">&nbsp;</span><span style="color: rgb(255, 0, 0); background-color: inherit;">unlock tables;</span>

mysql&gt;<span style="color: rgb(255, 0, 0); background-color: inherit;"><span style="background-color: inherit;">&nbsp;</span>set global read_only=0;</span>

&nbsp; &nbsp; &nbsp; 对于需要保证master-slave主从同步的salve库，如果要设置为只读状态，需要执行的命令为：

mysql&gt;<span style="background-color: inherit;">&nbsp;</span><span style="color: rgb(255, 0, 0); background-color: inherit;">set global read_only=1;</span>

&nbsp; &nbsp; &nbsp;将salve库从只读状态变为读写状态，需要执行的命令是：

mysql&gt;<span style="background-color: inherit;">&nbsp;</span><span style="color: rgb(255, 0, 0); background-color: inherit;">set global read_only=0;</span>

&nbsp; &nbsp; &nbsp;对于数据库读写状态，主要靠 &ldquo;read_only&rdquo;全局参数来设定；默认情况下，数据库是用于读写操作的，所以read_only参数也是0或faluse状态，这时候不论是本地用户还是远程访问数据库的用户，都可以进行读写操作；如需设置为只读状态，将该read_only参数设置为1或TRUE状态，但设置 read_only=1 状态有两个需要注意的地方：

&nbsp; &nbsp; &nbsp; 1.read_only=1只读模式，不会影响slave同步复制的功能，所以在MySQL slave库中设定了read_only=1后，通过 show slave status\G 命令查看salve状态，可以看到salve仍然会读取master上的日志，并且在slave库中应用日志，保证主从数据库同步一致；

&nbsp; &nbsp; &nbsp; 2.read_only=1只读模式，可以限定普通用户进行数据修改的操作，但不会限定具有super权限的用户的数据修改操作；在MySQL中设置read_only=1后，普通的应用用户进行insert、update、delete等会产生数据变化的DML操作时，都会报出数据库处于只读模式不能发生数据变化的错误，但具有super权限的用户，例如在本地或远程通过root用户登录到数据库，还是可以进行数据变化的DML操作；</span>

<span style="font-size: 18px; background-color: inherit;">&nbsp; &nbsp; &nbsp; 为了确保所有用户，包括具有super权限的用户也不能进行读写操作，就需要执行给所有的表加读锁的命令 &ldquo;flush tables with read lock;&rdquo;，这样使用具有super权限的用户登录数据库，想要发生数据变化的操作时，也会提示表被锁定不能修改的报错。</span>

<span style="font-size: 18px; background-color: inherit;">&nbsp; &nbsp; &nbsp; &nbsp; 这样通过 设置&ldquo;read_only=1&rdquo;和&ldquo;flush tables with read lock;&rdquo;两条命令，就可以确保数据库处于只读模式，不会发生任何数据改变，在MySQL进行数据库迁移时，限定master主库不能有任何数据变化，就可以通过这种方式来设定。</span>

<span style="font-size: 18px; background-color: inherit;">&nbsp; &nbsp; &nbsp; &nbsp;但同时由于加表锁的命令对数据库表限定非常严格，如果再slave从库上执行这个命令后，slave库可以从master读取binlog日志，但不能够应用日志，slave库不能发生数据改变，当然也不能够实现主从同步了，这时如果使用 &ldquo;unlock tables;&rdquo;解除全局的表读锁，slave就会应用从master读取到的binlog日志，继续保证主从库数据库一致同步。</span>

&nbsp;

<span style="font-size: 18px; background-color: inherit;">&nbsp; &nbsp; &nbsp; &nbsp;为了保证主从同步可以一直进行，在slave库上要保证具有super权限的root等用户只能在本地登录，不会发生数据变化，其他远程连接的应用用户只按需分配为select,insert,update,delete等权限，保证没有super权限，则只需要将salve设定&ldquo;read_only=1&rdquo;模式，即可保证主从同步，又可以实现从库只读。

&nbsp; &nbsp; &nbsp; &nbsp;相对的，设定&ldquo;read_only=1&rdquo;只读模式开启的解锁命令为设定&ldquo;read_only=0&rdquo;;设定全局锁&ldquo;flush tables with read lock;&rdquo;，对应的解锁模式命令为：&ldquo;unlock tables;&rdquo;.</span>

&nbsp;

<span style="font-size: 18px; background-color: inherit;">&nbsp; &nbsp; &nbsp; 当然设定了read_only=1后，所有的select查询操作都是可以正常进行的。</span>
