Tomcat 安装配置APR 模式

2016-08-22 tomcat tomcat

Tomcat可以使用APR来 提供超强的可伸缩性和性能，更好地集成本地服务器技术。APR(Apache Portable Runtime)是一个高可移植库，它是Apache HTTP Server 2.x的核心。APR有很多用途，包括访问高级IO功能(例如sendfile,epoll和OpenSSL)，OS级别功能(随机数生成，系统状态等 等)，本地进程管理(共享内存，NT管道和UNIX sockets)。这些功能可以使Tomcat作为一个通常的前台WEB服务器，能更好地和其它本地web技术集成，总体上让Java更有效率作为一个高 性能web服务器平台而不是简单作为后台容器。

按照官方说明需要：

&middot;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; APR library

&middot;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; OpenSSL libraries

tomcat-native 依赖于三个组件：APR, OPENSSL, JDK。

完整的APR实际上包含了三个开发包：apr、apr-util以及apr-iconv，每一个开发包分别独立开发，并拥有自己的版本。

apr中包含了一些通用的开发组件，包括mmap，DSO等等

apr-util该目录中也是包含了一些常用的开发组件。这些组件与apr目录下的相比，它们与apache的关系更加密切一些。比如存储段和存储段组，加密等等。

apr-iconv包中的文件主要用于实现iconv编码。目前的大部分编码转换过程都是与本地编码相关的。在进行转换之前必须能够正确地设置本地编码。因此假如两个非本地编码A和B需要转换，则转换过程大致为A-&gt;Local以及Local-&gt;B或者B-&gt;Local以及Local-&gt;A。

iconv是一个开源的字符编码转换库.

[https://apr.apache.org/download.cgi](https://apr.apache.org/download.cgi)

yum -y install apr-devel&nbsp; openssl openssl-devel libtool

预先部署jdk

1\. 安装apr-1.5.2.tar.gz

# tar zxvf apr-1.5.2.tar.gz

# cd apr-1.5.2

# ./configure

# make

# make install

//apr被安装在/usr/local/apr下

（如果上面安装过程出现/bin/rm: cannot remove `libtoolT&rsquo;: No such file or directory这个问题，

解决办法 vi configure 找到$RM &ldquo;$cfgfile&rdquo;这行注释掉，或者加一个-f ----$RM -f &quot;$cfgfile&quot;）

&nbsp;添加环境变量

# vi /etc/profile

添加：export LD_LIBRARY_PATH=/usr/local/apr/lib

echo &quot;export LD_LIBRARY_PATH=/usr/local/apr/lib&quot;&nbsp; &gt;&gt; /etc/profile

# source /etc/profile

2\. 安装apr-util-1.5.4.tar.gz

# tar zxvf apr-util-1.5.4.tar.gz

# cd apr-util-1.5.4

# ./configure --with-apr=/usr/local/apr

指定APR安装路径

# make

# make install

&nbsp;

3 安装apr-iconv-1.2.1.tar.gz ----可选

# ./configure --with-apr=/usr/local/apr

# make

# make install

4\. 安装tomcat-native.tar.gz

从Tomcat自带/bin目录中，解压tomcat-native.tar.gz文件：

# tar -zxvf tomcat-native.tar.gz

得到文件夹：tomcat-native-1.1.33-src

# cd tomcat-native-1.1.33-src/jni/native/

# ./configure --with-apr=/usr/local/apr&nbsp;&nbsp;&nbsp; （官网中例子的其他参数不需要，会自动找到）

# make &amp;&amp; make install

####

注意 最新版的tomcat自带Tomcat Native 1.2.8 需要OpenSSL 1.0.2h 版本

cd /opt/tomcat-native-1.2.8-src/native

先要更新系统的OpenSSL 1.0.2h 版本

# ./configure --with-apr=/usr/local/apr

# make &amp;&amp; make install
