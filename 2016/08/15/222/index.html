docker的基础命令及含义

2016-08-15 docker docker

一、基础命令

首先是docker的命令及含义

&nbsp;&nbsp;&nbsp; attach&nbsp;&nbsp;&nbsp; 进入一个容器

&nbsp;&nbsp;&nbsp; build&nbsp;&nbsp;&nbsp;&nbsp; 从Dockerfile编译

&nbsp;&nbsp;&nbsp; commit&nbsp;&nbsp;&nbsp; 提交容器

&nbsp;&nbsp;&nbsp; cp&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 从容器内复制文件到指定的路径上

&nbsp;&nbsp;&nbsp; create&nbsp;&nbsp;&nbsp; 创建容器

&nbsp;&nbsp;&nbsp; diff&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 可以列出容器内发生变化的文件和目录

&nbsp;&nbsp;&nbsp; events&nbsp;&nbsp;&nbsp; 从服务器拉取个人动态，可选择时间区间

&nbsp;&nbsp;&nbsp; exec&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 进入容器

&nbsp;&nbsp;&nbsp; export&nbsp;&nbsp;&nbsp; 将指定的容器保存成 tar 归档文件， docker import 的逆操作。导出后导入（exported-imported)）的容器会丢失所有的提交历史，无法回滚

&nbsp;&nbsp;&nbsp; history&nbsp;&nbsp;&nbsp; 查看指定镜像的创建历史

&nbsp;&nbsp;&nbsp; images&nbsp;&nbsp;&nbsp; 查看镜像列表

&nbsp;&nbsp;&nbsp; import&nbsp;&nbsp;&nbsp; 从归档文件（支持远程文件）创建一个镜像， export 的逆操作，可为导入镜像打上标签。导出后导入（exported-imported)）的容器会丢失所有的提交历史，无法回滚

&nbsp;&nbsp;&nbsp; info&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 显示系统信息

&nbsp;&nbsp;&nbsp; inspect&nbsp;&nbsp;&nbsp; 查看容器底层信息

&nbsp;&nbsp;&nbsp; kill&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 结束一个正在运行的容器

&nbsp;&nbsp;&nbsp; load&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 从 tar 镜像归档中载入镜像， docker save 的逆操作。保存后再加载（saved-loaded）的镜像不会丢失提交历史和层，可以回滚

&nbsp;&nbsp;&nbsp; login&nbsp;&nbsp;&nbsp;&nbsp; 登录Docker HUB

&nbsp;&nbsp;&nbsp; logout&nbsp;&nbsp;&nbsp; 退出Docker HUB

&nbsp;&nbsp;&nbsp; logs&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 打印出容器的标准输出

&nbsp;&nbsp;&nbsp; port&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 映射NAT的端口

&nbsp;&nbsp;&nbsp; pause&nbsp;&nbsp;&nbsp;&nbsp; 暂停容器

&nbsp;&nbsp;&nbsp; ps&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 列出容器

&nbsp;&nbsp;&nbsp; pull&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 下载一个镜像

&nbsp;&nbsp;&nbsp; push&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 将镜像推送至远程仓库，默认为 Docker Hub

&nbsp;&nbsp;&nbsp; restart&nbsp;&nbsp;&nbsp; 重启容器

&nbsp;&nbsp;&nbsp; rm&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 删除容器

&nbsp;&nbsp;&nbsp; rmi&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 删除镜像

&nbsp;&nbsp;&nbsp; run&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 运行一个新容器

&nbsp;&nbsp;&nbsp; save&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 保存镜像

&nbsp;&nbsp;&nbsp; search&nbsp;&nbsp;&nbsp; 搜索镜像

&nbsp;&nbsp;&nbsp; start&nbsp;&nbsp;&nbsp;&nbsp; 启动容器

&nbsp;&nbsp;&nbsp; stop&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 停止容器

&nbsp;&nbsp;&nbsp; tag&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 镜像改名

&nbsp;&nbsp;&nbsp; top&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 查看容器进程

&nbsp;&nbsp;&nbsp; unpause&nbsp;&nbsp;&nbsp; 恢复暂停的容器

&nbsp;&nbsp;&nbsp; version&nbsp;&nbsp;&nbsp; 查看版本

&nbsp;&nbsp;&nbsp; wait&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 阻塞对指定容器的其它调用方法，直到容器停止后退出阻塞

下面是docker run的命令及含义

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; -i&nbsp;&nbsp;&nbsp; #允许在容器进行交互

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; -t&nbsp;&nbsp;&nbsp; #分配伪tty终端

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; -d&nbsp;&nbsp;&nbsp; #在后台运行容器

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; -P&nbsp;&nbsp;&nbsp; #标示通知Docker所需的网络端口映射从主机映射到我们的容器内

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; -h&nbsp;&nbsp;&nbsp; #指定容器的hostname

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; -e&nbsp;&nbsp;&nbsp; #设置容器的环境变量

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; -a&nbsp;&nbsp;&nbsp; #stdin 指定标准输入输出内容类型，可选 STDIN/STDOUT / STDERR 三项

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; -w&nbsp;&nbsp;&nbsp; #指定工作目录

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; -u&nbsp;&nbsp;&nbsp; #指定用户名id

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; -m&nbsp;&nbsp;&nbsp; #指定内存

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; --env-file=[]&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 从指定文件读入环境变量

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; --restart=always&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 当容器推出时，自动重启容器

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; --restart=on-failure:5&nbsp;&nbsp;&nbsp; 当容器推出代码为非0时，docker会尝试自动重启该容器，最多重启5次

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; --name&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 指定容器名，容器名在整个docker环境中必须唯一

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; --dns&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 指定dns

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; --cpuset=&quot;&quot;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 绑定容器到指定CPU运行

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; --net=&quot;bridge&quot;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 指定容器运行的网络模式
