    #
    1. 写在前头 

    从甲方的角度看看，如何进行数据中心机柜综合布线改造&nbsp;

本文仅仅从数据中心客户自身为出发点，对租用数据中心中的机柜进行按需改造的详细介绍，并非数据中心机房基础建设

    #
    2. 简介

    数据中心机房建设相比于传统的园区网，由于服务的多元化、业务无规律增加等问题，所以除了具备安全性、健壮性特点外，还需要支持较高的扩容能力和灵活的按需变化能力

    3. 前期知识储备

    ##
    3.4 布线方法

    ###
    3.4.1 EOR(End of Row)

    ![](http://read.html5.qq.com/image?src=forum&amp;q=5&amp;r=0&amp;imgflag=7&amp;imageUrl=http://mmbiz.qpic.cn/mmbiz_png/JeosubumlZqibweS21TFuZgWXTJqZpEhzCd0hHawajerChFryI239vS4NTJZ8jK8NMembj1vLuDCxWlLlgoEQ9A/0?wx_fmt=png)

<span style="margin: 0px; padding: 0px; line-height: 24px; font-family: Helvetica, &quot;Hiragino Sans GB&quot;, 微软雅黑, &quot;Microsoft YaHei UI&quot;, SimSun, SimHei, arial, sans-serif; font-size: 15px; -ms-word-wrap: break-word !important; max-width: 100%; box-sizing: border-box !important;">每个机柜组中的两排机柜的最边端机柜作为网络列头柜，在网络列头柜及服务器机柜上方会安装配线架（由于会增加故障点，所以现在很多的互联网公司会放弃配线架）</span>

    ###
    3.4.2 MOR(Middle of Row)

    ###

    <span style="margin: 0px; padding: 0px; line-height: 24px; font-size: 15px; -ms-word-wrap: break-word !important; max-width: 100%; box-sizing: border-box !important;">相对于EOR模式，MOR只是把网络列头柜放置在一排机柜的中间位置，减少水平端布线成本（通常是铜缆）</span>

    ###
    3.4.3 TOR(Top of Rack)

    ###

    <span style="margin: 0px; padding: 0px; line-height: 24px; font-size: 15px; -ms-word-wrap: break-word !important; max-width: 100%; box-sizing: border-box !important;">采用TOR布线时，机柜组中的每个服务器机柜上方部署1-2台接入层交换机，内部的服务器直接通过跳线接入到机柜上方的交换机，交换机上行端口通过光纤接入到核心交换机（汇聚交换机）</span>

    ###
    3.4.4 不同布线方法的利弊

    ● EOR/MOR&nbsp;

优点：有利于提高交换的端口利用率，降低设备平均端口能耗&nbsp;

缺点：水平电缆长度更长，桥架压力大

● TOR&nbsp;

优点：缩短了铜缆的使用距离，简化了线缆管理；增加光纤的使用替代铜缆，减少了网络拥塞，可扩展性强，可以直接复制&nbsp;

缺点：网络设备端口资源浪费，主干光纤使用量增加，对于核心交换机的端口需求压力大

    #
    4. 光纤技术发展发展趋势

    <pre style="margin: 10px 0px; padding: 6px 10px; border-radius: 3px; border: 1px solid rgb(204, 204, 204); border-image: none; color: rgb(62, 62, 62); text-transform: none; line-height: 19px; text-indent: 0px; letter-spacing: normal; overflow: auto; font-family: Consolas, &quot;Liberation Mono&quot;, Menlo, Courier, monospace; font-size: 13px; font-style: normal; font-weight: normal; word-spacing: 0px; -ms-word-wrap: break-word; max-width: 100%; box-sizing: border-box !important; orphans: 2; widows: 2; background-color: rgb(255, 255, 255); -webkit-text-stroke-width: 0px; font-variant-ligatures: normal; font-variant-caps: normal; text-decoration-style: initial; text-decoration-color: initial;">

    布线，特别是数据中心布线世界是属于铜缆的，也是属于光纤的，但是归根结底是属于光纤的 &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;&nbsp;--- 孙慧永：罗森伯格公司高级产品经理

    ● 带宽越来越高&nbsp;

随着云计算的突飞猛进，带宽将会越来越高，万兆已经成为必然

● 密度&nbsp;

绿色机房已经越来越流行，Facebook、Google、Appl都建设了绿色机房，即使我们目前还没达到那个程度，但是提高密度讲有效的提高机房利用率，降低端口的平均能耗

● 可靠性&nbsp;

需要租用数据中心的（或者自建）设备，想必其业务的重要性，意外断网的损失是巨大的，必须要超级高的可用性保证才行

● 管理&nbsp;

数据中心的运营成本正在持续上升，我们需要更为简易的管理方式来降低管理成本

    #
    5. 布线规范

    ● 远程管理网线&nbsp;

远程管理网线使用超五类网线，从机柜左边（从机柜正面看）下放，并且长度延伸至层板右边即可（不得超出），在层板中间位置用扎带固定。另外备线整理成一圈，固定在机柜中间位置，确保能够到达该机柜所有位置。

● 电源线&nbsp;

当数据中心使用的是交流220V电源（直接接入排插），将电源线从机柜左边（从机柜正面看）下放，并且长度延伸至层板右边（不得超出），在层板中间位置使用扎带固定。&nbsp;

当数据中心使用的是高压直流电源（需要将电源线剥皮接入空开），将电源线从机柜左边（从机柜正面看）下放，并且长度延伸至层板右边（不得超出），在层板中间位置使用扎带固定。接口端剥皮接入空开，地线使用铜鼻子接入地线铜线排

● 内网网线&nbsp;

内网网线使用六类成品跳线，从机柜右边（从机柜正面看）下放，并且长度延伸至层板左边即可（不得超出），在层板中间位置使用扎带固定。另外备线整理成一圈，固定在机柜中间位置，确保能够到达该机柜所有位置。

● 外网网线&nbsp;

外网网线使用六类成品跳线 ，从机柜右边（从机柜正面看）下放，并且长度延伸至层板左边即可（不得超出），在层板中间位置使用扎带固定。另外备线整理成一圈，固定在机柜中间位置，确保能够到达该机柜所有位置。

    #
    6. 交换机端口分配规范

    交换机端口使用从端口1开始，对应机柜号小的机柜，以此类推，备线使用端口号大的交换机端口；例如：端口1-16分配给机柜1（15、16作为备线），端口17-32分配给机柜2（31、32作为备线），端口33-48分配给机柜3（47、48作为备线）

原则：交换机端口利用率尽量高，并且确保服务器高可用性要求，同一机架位若接两根网线，必须接入不同交换机

    #
    7. 标签张贴规范

    网络维护管理者的基本工作就是保证网络正常运行。如果没有做相应的标识，或者标识不规范，当网络发生问题时，如何在数千甚至数万根线缆中找到发生问题的具体位置呢？

● 网线标签&nbsp;

按照标签标准进行打印，长度使用95mm，张贴在网线水晶头内13cm处，并且两头都要张贴

● 光纤标签&nbsp;

按照标签标准进行打印，长度使用90mm，张贴在光纤头内10cm处，并且两头都要张贴

● 电源线标&nbsp;

签按照现场实际情况进行打印，张贴在电源线接服务器端13cm处，另一头贴在空开位置或插座位置。

    #
    8. 网线购买规范

    ● 内外网网线使用米数、数量根据现场测量后进行统一订制，必须使用六类成品网线（机器压制水晶头）

● 远程管理网线使用米数、数量根据现场测量后进行统一订制，必须使用超五类及以上成品网线（机器压制水晶头）

● 所有购买的网线都必须使用FLUKE测试后给出测试报告（PDF文档）

    #
    9. 工程验收

    数据中心的运营成本在持续上升，每话费1美元购置新的基础设施，就得话费8美元开展维护和管理

● 外观（层板、线材位置）&nbsp;

美观很重要，人都是视觉动物，需要在美观与实用之间寻找一个平衡点

● 电源&nbsp;

电源从列头柜到物理服务器，中间跳转好几级，我们更多的需要测试接入空开是否正常

● 网络连通（铜缆、光纤）&nbsp;

布线过程对铜缆、光纤的拉扯容易导致线缆连通异常，需要在调试好网络设置后，进行抽查（当然有时间可以全部测试），正常布线之前会要求综合布线的乙方进行网线的fluke测试，以及光纤的光衰测试

● 标签&nbsp;

标签是命门！我是栽过跟头的，后期维护上，现场的标签是非常重要的，打印设备、线缆标识、配线架标识、面板标识、设备管理标识、标识文档等等都是相当重要的！&nbsp;

为了最佳读取效果，所有的标签应由机械设备打印,不应采用手写方式

    #
    10. 方案编写

    很多在互联网公司在计划进行数据中心机柜综合布线时都是提需求，然后由乙方来提交方案，但是经常由于双方编写方案的出发点不同，导致很多需求不对称，所以可以自己编写方案，并严格要求乙方按照方案执行！当然施工过正常最重要的是需求沟通，我做过的数据中心机柜改造项目中，70%以上的时间都在做需求沟通！！&nbsp;

如果你想从甲方的角度编写一个方案，需要做的东西如下

● 现场平面图&nbsp;

IDC机房平面图，找运营商要电子档（必要时候现场确认）&nbsp;

机房整体确认：一排几个机柜；每个机柜几U；电源接口几个（空开、排插接口）

● 网络布线示意图

● 网线接入端口详细说明

● 光纤布线接入位置详细说明

● 电源线接入方式说明

● 标签打印说明

● 标签粘贴规范说明

● 层板调整规范说明

● 材料（自行准备、乙方准备、机房准备）

● 报价表单

● 投标确认乙方

● 如何进场&nbsp;

数据中心机房通常都是租用运营商机房，所以进场都需要提前申请好，项目施工时间段；进场人员；证件；进场材料清单；出场材料清单

● 需求沟通
