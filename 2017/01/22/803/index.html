    linux部署tmpfs缓存

    linux linux
    创建时间:2017-01-22 15:58

    tmpfs是Linux/Unix系统上的一种基于内存的虚拟文件系统。

    &nbsp;Centos 7 中的一个文件系统tmpfs，这是一个将所有文件和文件夹写到虚拟内存中而不是实际写到磁盘中的虚拟文件系统。这意味中tmpfs中所有的内容都是临时的，在tmpfs卸载、系统重启或者电源切断后内容都将会丢失。

技术的角度上来说，tmpfs将所有的内容放在内核内部缓存中并且会调整大小来容纳文件，并可从交换空间中交换出不需要的页。

Linux默认(CentOS)/dev/shm分区的大小是系统物理内存的50%

tmpfs可以使用您的内存或swap分区来存储文件(即它的存储空间在virtual memory 中, VM由real memory和swap组成)。

由此可见，tmpfs主要存储暂存的文件。它有如下2个优势 : 1\. 动态文件系统的大小。2\. tmpfs 使用VM建的文件系统，速度当然快。3.重启后数据丢失。

当删除tmpfs中的文件时,tmpfs会动态减少文件系统并释放VM资源,LINUX中可以把一些程序的临时文件放置在tmpfs中，利用tmpfs比硬盘速度快的特点提升系统性能。

实际应用中，为应用的特定需求设定此文件系统，可以提升应用读写性能，如将squid 缓存目录放在/tmp, php session 文件放在/tmp, socket文件放在/tmp, 或者使用/tmp作为其它应用的缓存设备

/dev - 含有针对所有设备的设备文件的目录

/dev/shm - 包含共享内存分配

/run - 用于系统日志

/sys/fs/cgroup - 用于cgrpups， 一个针对特定进程限制、管制和审计资源利用的内核特性

临时修改/dev/shm大小

#mount -o size=1500M -o nr_inodes=1000000 -o noatime,nodiratime -o remount /dev/shm

一般修改大小

umount /dev/shm

mount -t tmpfs thmfs -o size=16g /dev/shm

用法一：

挂载tmpfs:

mount&nbsp; -t tmpfs -o size=分配空间大小 tmpfs自定义名称 挂载位置

例：

mkdir /tmpfs

mount -t tmpfs -o size=1024m my_tmpfs /tmpfs

&nbsp;

也可以动态调整tmpfs空间大小：

mount 挂载位置 -o remount,size=重新分配空间大小

例：

mount /tmpfs -o remount,size=2048m

&nbsp;

卸载已挂载的tmpfs

umount /tmpfs

用法二：

首先在/dev/shm建个tmp文件夹，然后与实际/tmp绑定（则/tmp 即使用tmpfs文件系统）

mkdir /dev/shm/tmp

chmod 1777 /dev/shm/tmp

mount --bind /dev/shm/tmp /tmp

&nbsp;卸载umount /dev/shm/tmp

最后开机启用的配置

可以在vi /etc/fstab 中定义其大小

tmpfs /dev/shm tmpfs,defaults,size=2048m 0 0

tmpfs /tmp tmpfs defaults,size=1024m 0 0

修改后执行mount -o remoount /dev/shm 后生效
